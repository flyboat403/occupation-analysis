## 二、核心模板

> 以下模板是 Agent 实际需要的核心内容，应优先参考。

### 2.1 行动领域描述通用模板

| 要素 | 描述内容 |
|------|----------|
| **行动领域名称** | 例如："简单机械零件的数控车削加工" |
| **典型工作任务描述** | 在[某类企业/部门]中，[谁]接到[某类任务（如生产订单/客户需求）]，需要[独立或协作]完成[具体工作内容]，最终交付[成果/产品/服务]，并满足[质量标准/时间要求/安全规范]。 |
| **工作过程** | 1. **信息**：识读图纸/工艺文件，明确加工要求。<br>2. **计划**：选择刀具、量具，确定加工步骤。<br>3. **决策**：确定切削参数，优化加工顺序。<br>4. **执行**：编写/调试程序，装夹工件，进行加工。<br>5. **检查**：自检关键尺寸，判断是否符合图纸。<br>6. **评估**：分析废品原因，提出改进措施。 |
| **工作与学习条件** | - 工具设备：数控车床、计算机（CAD/CAM软件）、游标卡尺、千分尺等。<br>- 劳动组织：一般独立完成，复杂件可2-3人小组协作。<br>- 资料：零件图、工艺卡、设备操作手册、安全规程。 |
| **能力描述** | - **专业能力**：读懂机械图纸，熟练操作数控车床，会编程及选择刀具。<br>- **方法能力**：能查阅手册，能独立规划工作步骤，能进行质量自检。<br>- **社会能力**：遵守安全规程，能与同事协作，能清晰记录加工过程。 |
| **能力等级** | 初级工/中级工/高级工/技师 |
| **获得途径** | 系统学习→实训操作→考取职业资格证书 |

### 2.2 学习领域描述通用模板

| 要素 | 描述内容 |
|------|----------|
| **学习领域编号与名称** | 例如："C01 汽车发动机检修" |
| **参考学时** | 通常48-96学时（中职常用64学时，高职常用96学时） |
| **学期安排** | 建议安排在第几学期（体现由简到难的课程排序） |
| **学习目标（能力描述）** | 学生完成本学习领域后，**能够**做什么（综合能力）。应包含专业、方法、社会能力。格式为"能……"。 |
| **学习内容（内容描述）** | 为实现上述能力，学生需要学习哪些具体知识、技能、工作方法。通常按工作过程的六个步骤来组织内容。 |
| **教学方法** | 建议的教学方法（如项目教学、任务驱动、理实一体化）、教学场地等。 |
| **评价方式** | 考核方式（如过程考核60%+终结考核40%）。 |

### 2.3 学习情境描述通用模板

| 要素 | 描述内容 |
|------|----------|
| **学习情境名称** | 具体、形象，能概括任务核心。例如："卡罗拉轿车左前轮刹车异响的检修" |
| **参考学时** | 完成该情境所需的教学时间，通常8-32学时 |
| **学习目标** | 学生完成该情境后**具体能做什么**。需具体、可测量。 |
| **学习内容** | 为实现目标，学生需要掌握的知识、技能、方法（按工作过程组织）。 |
| **教学方式** | 教学场地、工具清单、教学方法建议（如小组分工）、考核方式。 |

## 三、核心理念

### 3.1 工作过程导向

课程内容来源于真实的工作过程，强调：

- **工作对象**：工作的物质或非物质对象
- **工作内容**：具体的工作任务
- **工作过程**：按照"信息-计划-决策-执行-检查-评估"六步骤
- **工作手段**：工具、设备、技术
- **工作组织**：工作流程和组织形式
- **工作成果**：工作产品或服务
- **工作能力**：完成工作所需的专业能力、方法能力、社会能力

### 3.2 典型工作任务

典型工作任务（Typical Work Task）是职业工作过程中具有代表性、能反映职业本质特征的工作任务。

**特征**：
- 具有完整的工作过程（准备、实施、评价）
- 体现职业的典型工作内容
- 具有一定的难度和挑战性
- 能促进职业能力发展

**工作任务分析要素**：
| 要素 | 说明 | 示例 |
|------|------|------|
| 工作对象 | 工作的物质或非物质对象 | 海报、VI、插画等视觉作品 |
| 工具/材料/设备 | 完成工作所需的工具和设备 | Photoshop、Illustrator、数位板等 |
| 工作方法 | 完成工作的流程和步骤 | 调研了解需求→素材采集→设计制作→成品输出 |
| 劳动组织方式 | 工作的组织形式 | 独立完成或小组协作 |
| **工作要求** | **完成工作需达到的标准** | **符合设计规范，创意独特，视觉效果良好，按时完成** |

**⚠️ MUST DO：工作要求字段必须填写具体内容，不可空白**

工作要求应包含：
- 质量标准（如：符合行业规范、达到XX标准）
- 时间要求（如：按时完成、在XX小时内完成）
- 技术要求（如：精度达到XX、效果达到XX）
- 安全要求（如：遵守安全规程）

### 3.3 行动领域

行动领域（Action Field）是工作任务的集合，按照工作对象、工作难度等维度归类形成。

**划分原则**：
- 同类工作对象归为一类
- 相似工作难度归为一类
- 考虑工作过程完整性

**⚠️ MUST DO：聚类原则字段必须填写具体内容，不可空白**

聚类原则应包含：
- 工作对象相似性说明（如：工作对象相似（视觉作品））
- 难度梯度说明（如：难度梯度（初级→中级→高级））
- 工作逻辑连续性说明（如：工作逻辑连续（UI→交互→网页））
- 技术领域一致性说明（如：技术领域一致（VR/AR））

**聚类原则示例**：
| 行动领域 | 聚类原则 |
|----------|----------|
| 数字视觉设计 | 工作对象相似（视觉作品），难度梯度（初级→高级） |
| 用户界面与交互设计 | 工作对象相似（界面与交互），工作逻辑连续（UI→交互→网页） |
| 数字音视频制作 | 工作对象相似（音视频内容），工作流程连续（采集→编辑） |
| 三维动画与特效制作 | 工作对象相似（三维与特效），难度梯度（中级→高级） |

### 3.4 学习领域

学习领域（Learning Field）是由行动领域转化而来的课程单元，是学校教学的基本单位。

**转换原则**：
- 保持工作过程完整性
- 遵循学习认知规律
- 考虑教学可行性

## 四、开发流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  工作任务分析  │ ──▶ │ 典型工作任务  │ ──▶ │  行动领域    │ ──▶ │  学习领域    │
│              │     │    确定      │     │    划分      │     │    转换      │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
       │                    │                    │                    │
       ▼                    ▼                    ▼                    ▼
   调研企业              筛选代表性            归类工作任务          设计课程
   岗位分析              任务                 形成领域              方案
```

### 步骤1：工作任务分析

**方法**：
1. 企业调研
2. 岗位分析
3. 工作任务采集
4. 任务描述

**产出**：各岗位的工作任务列表

### 步骤2：确定典型工作任务

**筛选标准**：
1. 代表性：反映职业典型工作内容
2. 完整性：包含完整工作过程
3. 教育性：能促进学生能力发展
4. 可行性：学校可实施教学

**产出**：典型工作任务汇总表

### 步骤3：行动领域划分

**聚类维度**：
- 按工作对象：人、物、信息等
- 按工作难度：初级、中级、高级
- 按工作过程：准备、实施、管理

**产出**：行动领域表

#### 行动领域划分一致性检查

> **关键问题**：大模型生成的行动领域可能出现不一致，需要验证并修正。

**常见不一致类型**：

| 类型 | 表现 | 诊断方法 | 处理方式 |
|------|------|----------|----------|
| **任务归属冲突** | 同一任务被分配到多个行动领域 | 检查typical_tasks中每个任务的action_domain归属 | 重新归类，确保任务唯一归属 |
| **领域边界模糊** | 相邻领域任务内容重叠 | 比较领域间的任务列表，检查是否有相同关键词 | 明确边界，重新划分或合并 |
| **难度梯度断裂** | 领域内任务难度跨度过大或无明显梯度 | 分析领域内任务的difficulty_level | 重新划分难度梯度，或拆分领域 |
| **工作对象混乱** | 同一领域包含多种工作对象 | 检查领域内任务的work_object字段 | 按工作对象重新归类 |

**一致性验证流程**：

```
步骤1：检查任务归属唯一性
       ├─ 每个典型任务应唯一归属一个行动领域
       ├─ 发现重复归属 → 分析任务本质 → 确定主要归属
       └─ 删除次要归属
       
步骤2：检查领域边界清晰性
       ├─ 比较相邻领域的任务列表
       ├─ 发现重叠 → 分析重叠任务的核心特征
       └─ 根据核心特征重新归类
       
步骤3：检查难度梯度合理性
       ├─ 分析每个领域内任务的难度分布
       ├─ 梯度断裂 → 考虑拆分领域或重新分配任务
       └─ 梯度合理 → 验证通过
       
步骤4：检查工作对象一致性
       ├─ 分析领域内任务的工作对象
       ├─ 发现多种对象 → 分析是否为同一工作过程的不同阶段
       └─ 若非同一过程 → 重新归类
```

**处理示例**：

```
问题：发现"汽车发动机检修"任务同时归属"发动机检修领域"和"故障诊断领域"

诊断分析：
- 任务本质：检修是故障诊断后的执行环节
- 主要归属：发动机检修领域（执行阶段）
- 次要归属：故障诊断领域（决策阶段）

处理方式：
- 将"汽车发动机检修"任务明确归属"发动机检修领域"
- 在"故障诊断领域"保留"发动机故障诊断"任务（决策阶段）
- 确保领域边界清晰：诊断（决策）→ 检修（执行）
```

### 步骤4：学习领域转换

**转换要点**：
1. 教学目标设计
2. 教学内容选择
3. 教学方法设计
4. 教学评价设计

**产出**：学习领域描述表

## 五、学习领域推导算法

### 5.1 学习领域数量计算

**从行动领域推导学习领域数量**：

| 行动领域数量(N) | 学习领域数量计算规则 |
|-----------------|----------------------|
| N ≤ 4 | 学习领域数量 = N（一对一映射） |
| N = 5-7 | 学习领域数量 = N 或 N+1（复杂领域可拆分） |
| N ≥ 8 | 学习领域数量 ≈ N × 1.2（部分领域拆分为2个） |

**教育层次数量要求**：
- 中职：学习领域数量 ≥ 8个（建议8-10个）
- 高职：学习领域数量 ≥ 10个（建议10-12个）
- 职教本科：学习领域数量 ≥ 10个（建议12-15个）

### 5.2 学习领域命名规则

**编号格式**：`LD{序号}` 或 `C{序号}`
- 例如：LD01, LD02, ... 或 C01, C02, ...

**名称组合模式**：`[工作对象] + [核心工作动词]`

| 组合要素 | 说明 | 示例 |
|----------|------|------|
| **工作对象** | 行动领域处理的对象 | 服装、网络设备、汽车发动机 |
| **核心工作动词** | 主要工作活动 | 设计、制作、检修、配置 |

**命名示例**：
- 行动领域："服装款式创意设计" → 学习领域："服装款式设计"
- 行动领域："网络设备的安装与调试" → 学习领域："网络设备配置"
- 行动领域："汽车发动机故障诊断与维修" → 学习领域："汽车发动机检修"

### 5.3 行动领域→学习领域拆分/合并判断

> **核心思想**：拆分/合并决策应基于**教学可行性**和**学生学习效果**分析，而非机械条件判断。

#### 5.3.1 拆分判断依据（语义分析）

**拆分决策流程**：

```
步骤1：分析行动领域的教学可行性
       ├─ 领域内任务数量是否超过教学容量？
       │   └─ 判断标准：单领域典型任务>5个 → 拆分可能性高
       │   └─ 但需进一步分析任务关联性
       │
       ├─ 领域内工作对象是否差异过大？
       │   └─ 判断标准：工作对象类型>2种 → 拆分可能性高
       │   └─ 但需分析对象是否为同一工作过程的不同阶段
       │
       └─ 领域内任务难度跨度是否过大？
           └─ 判断标准：同时包含"简单"和"复杂"任务 → 拆分可能性高
           └─ 但需分析是否体现学习梯度

步骤2：评估拆分后的教学效果
       ├─ 拆分是否有利于学生循序渐进学习？
       ├─ 拆分是否导致领域间内容重复？
       └─ 拆分是否导致单领域学时过低（<48学时）？

步骤3：综合判断并决策
       ├─ 教学可行 + 教学效果改善 → 决定拆分
       ├─ 教学可行但效果不改善 → 保持原状
       └─ 教学不可行 → 必须拆分
```

**拆分场景示例分析**：

| 场景 | 任务分析 | 工作对象分析 | 教学效果预测 | 决策 |
|------|----------|-------------|--------------|------|
| **服装制作** | 包含5个任务：缝制、裁剪、整烫、质检、包装 | 工作对象：服装（同一对象的不同工序）| 若拆分：缝制+裁剪为"服装工艺"，整烫+质检+包装为"服装后处理"，可能导致学时不均衡 | **不拆分**（工作过程完整性更重要）|
| **服装设计与制作** | 包含"设计"和"制作"两类任务 | 工作对象：设计图纸（设计阶段）+ 服装实体（制作阶段）| 若拆分：设计独立领域，制作独立领域，学生可先学设计再学制作，符合认知梯度 | **拆分**（工作对象差异大，体现学习梯度）|
| **网络运维** | 包含"网络配置"和"故障排除"两类任务 | 工作对象：网络设备（同一对象）| 若拆分：配置领域（初级技能），故障排除领域（高级技能），学生先学配置再学故障排除，符合能力梯度 | **拆分**（难度跨度大，体现能力梯度）|

**拆分后的命名建议**：

```
拆分命名原则：
├─ 保持工作过程关联性 → 名称体现上下游关系
├─ 体现难度梯度 → 初级领域名称简洁，高级领域名称具体
└─ 避免内容重复 → 名称明确区分领域边界

示例：
- "服装设计与制作" 拆分 → "服装款式设计" + "服装缝制工艺"
- "网络运维" 拆分 → "网络设备基础配置" + "网络故障诊断与排除"
- "汽车维修" 拆分 → "汽车常规维护" + "汽车故障诊断与排除"
```

#### 5.3.2 合并判断依据（语义分析）

**合并决策流程**：

```
步骤1：分析两个行动领域的关联性
       ├─ 工作对象是否相同？
       │   └─ 判断标准：处理同一对象的不同阶段 → 合并可能性高
       │
       ├─ 任务是否存在强依赖关系？
       │   └─ 判断标准：任务A的输出是任务B的输入 → 合并可能性高
       │
       └─ 合并后的学时是否合理？
           └─ 判断标准：合并后学时≤144学时 → 合并可行性高

步骤2：评估合并后的教学效果
       ├─ 合并是否有利于学生理解完整工作过程？
       ├─ 合并是否导致领域内容过于庞杂？
       └─ 合并是否导致领域学时过高（>144学时）？

步骤3：综合判断并决策
       ├─ 教学效果改善 + 学时合理 → 决定合并
       ├─ 教学效果改善但学时过高 → 保持原状
       └─ 教学效果不改善 → 保持原状
```

**合并场景示例分析**：

| 场景 | 关联性分析 | 学时分析 | 教学效果预测 | 决策 |
|------|-----------|----------|--------------|------|
| **服装材料选用 + 服装款式设计** | 工作对象不同（材料→设计图纸），但材料选用是设计的前置知识 | 若合并：预计48+48=96学时，合理 | 合合有利于学生理解"材料→设计"的完整流程 | **合并**（强依赖关系）|
| **设备安装 + 设备调试** | 工作对象相同（设备），安装是调试的前置步骤 | 若合并：预计32+48=80学时，合理 | 合合有利于学生理解"安装→调试"的完整过程 | **合并**（工作过程完整性）|
| **汽车维修 + 汽车销售** | 工作对象不同（汽车维修→客户），任务无依赖关系 | 若合并：预计72+48=120学时，虽合理但内容庞杂 | 合并导致领域内容过于庞杂，不利于教学 | **不合并**（关联性弱）|

**合并后的命名建议**：

```
合并命名原则：
├─ 体现完整工作过程 → 名称涵盖合并前两个领域的关键动作
├─ 保持简洁性 → 名称不超过10个字
└─ 避免歧义 → 名称明确表达领域边界

示例：
- "服装材料选用" + "服装款式设计" 合并 → "服装设计基础"
- "设备安装" + "设备调试" 合并 → "设备安装调试"
- "网络配置" + "网络维护" 合并 → "网络运维"
```

#### 5.3.3 拆分/合并决策对照表

| 决策 | 关键判断依据 | 教学效果改善 | 典型场景 |
|------|-------------|--------------|----------|
| **拆分** | 工作对象差异大 / 难度跨度大 / 任务数量>5且任务独立性强 | 有利于学生循序渐进学习 | 设计+制作、配置+故障排除 |
| **合并** | 工作对象相同 / 任务强依赖 / 合并后学时≤144 | 有利于学生理解完整工作过程 | 材料选用+设计、安装+调试 |
| **保持** | 任务关联适中 / 学时已合理 / 拆分/合并均无改善 | 保持当前结构更清晰 | 单一工作过程的领域 |

### 5.4 学时估算指导原则

> **核心思想**：学时应基于任务的**真实认知负荷**和**教学实际需求**进行语义分析，而非机械的系数相乘。

#### 5.4.1 核心判断依据

**1. 任务复杂度分析**（认知负荷视角）

| 任务类型 | 认知负荷特征 | 预计学时范围 | 典型示例 |
|----------|-------------|--------------|----------|
| **简单任务** | 单一操作流程，无需决策判断 | 16-32学时 | 设备日常保养、零部件更换 |
| **中等任务** | 多步骤操作，需要简单判断和选择 | 32-48学时 | 故障初步诊断、参数调整 |
| **复杂任务** | 综合决策，需要信息整合和创新思考 | 48-72学时 | 工艺方案优化、复杂故障排除 |

**判断方法**：
- 分析每个典型任务的**决策节点数量**（越多越复杂）
- 评估任务的**信息处理深度**（识别→理解→应用→分析→评估→创造）
- 考虑任务的**错误后果**（错误后果越严重，需更多训练时间）

**2. 教学方式需求**

| 教学方式 | 学时需求特点 | 理论与实践比例建议 |
|----------|-------------|-------------------|
| **项目教学/任务驱动** | 需要完整项目周期，学时较多 | 理论30% + 实践70% |
| **理实一体化** | 理论与实践交替，学时适中 | 理论40% + 实践60% |
| **讲授式/演示式** | 信息传递为主，学时较少 | 理论50% + 实践50% |

**判断方法**：
- 根据任务性质选择教学方式（操作类→项目教学，知识类→讲授）
- 项目教学领域学时增加20-30%

**3. 教育层次适配**

| 层次 | 学时分配特点 | 核心领域学时建议 |
|------|-------------|------------------|
| **中职** | 操作训练为主，实践学时占比高 | 64-96学时（实践≥60%）|
| **高职** | 技术应用与诊断，理论与实践均衡 | 72-120学时（实践50-60%）|
| **职教本科** | 系统优化与创新，理论学时占比提高 | 96-144学时（理论≥40%）|

**判断方法**：
- 中职：强调操作熟练度→重复训练→学时可略多
- 高职：强调诊断推理→案例分析→学时适中
- 职教本科：强调系统思维→项目综合→学时较多

**4. 学校条件约束**

| 条件因素 | 对学时的影响 | 调整建议 |
|----------|-------------|----------|
| **设备台套数充足** | 可小组同时操作，等待时间少 | 学时可压缩10-15% |
| **设备台套数不足** | 需轮换教学，等待时间长 | 学时需增加15-20% |
| **师资经验丰富** | 教学效率高，时间利用率高 | 学时可压缩5-10% |
| **师资新手** | 教学节奏慢，需要更多指导 | 学时需增加10-15% |

#### 5.4.2 学时估算流程

```
步骤1：分析每个典型任务的真实复杂度
       ├─ 判断决策节点数量
       ├─ 评估信息处理深度
       └─ 考虑错误后果严重性
       
步骤2：根据任务性质选择教学方式
       ├─ 操作类任务 → 项目教学（学时增加）
       ├─ 知识类任务 → 讲授式（学时减少）
       └─ 混合类任务 → 理实一体（学时适中）
       
步骤3：估算单领域学时
       ├─ 汇总领域内各任务复杂度
       ├─ 叠加教学方式需求
       └─ 叠加教育层次需求
       
步骤4：汇总并验证总学时
       ├─ Σ(各学习领域学时)
       ├─ 对比教育层次最低要求
       └─ 不足则增加核心领域学时
```

#### 5.4.3 学时调整优先级

| 优先级 | 调整方式 | 适用场景 | 调整幅度 |
|--------|----------|----------|----------|
| **1** | 增加核心学习领域学时 | 核心技能领域学时偏低（<48）| 增加16-32学时 |
| **2** | 添加拓展学习领域 | 总学时差距较大（>200学时）| 新增1-2个领域（48-72学时）|
| **3** | 提高复杂任务领域学时 | 复杂任务训练不足 | 增加8-16学时 |
| **4** | 调整情境学时（微调）| 总学时差距较小（<50学时）| 每情境±4-8学时 |

#### 5.4.4 常见学时估算场景

**场景A：简单操作类学习领域**
```
典型任务："设备日常保养"
- 任务复杂度：简单（单一流程，无决策）
- 教学方式：项目教学（学生动手操作为主）
- 教育层次：中职（强调操作熟练）
- 估算：16学时（基准）× 2（项目教学系数）× 1.2（熟练训练）≈ 32-48学时
```

**场景B：复杂诊断类学习领域**
```
典型任务："发动机故障诊断与排除"
- 任务复杂度：复杂（多决策节点，信息整合）
- 教学方式：项目教学+案例分析
- 教育层次：高职（诊断推理能力）
- 估算：48学时（复杂任务基准）× 1.3（项目教学）× 1.1（高职层次）≈ 64-96学时
```

**场景C：系统设计类学习领域**
```
典型任务："生产工艺优化"
- 任务复杂度：复杂（创新思考，系统整合）
- 教学方式：项目教学+小组研讨
- 教育层次：职教本科（系统优化能力）
- 估算：48学时（复杂任务基准）× 1.5（项目+研讨）× 1.2（本科层次）≈ 96-120学时
```

## 六、情境设计方法论

### 6.1 学习情境数量计算

### ⚠️ MANDATORY - 学习情境生成规则

> **新增学习领域时必须同步生成学习情境，不得遗漏**

**生成规则**：
- 每个学习领域必须生成 **3-6个学习情境**
- 新增学习领域时，同步生成配套学习情境（不可后续补充）
- 学习情境名称必须具体化，符合命名规则

**常见遗漏场景**：
| 场景 | 问题 | 正确做法 |
|------|------|----------|
| 拆分学习领域 | 只拆分领域，未生成情境 | 拆分同时为新领域生成3-4个情境 |
| 新增拓展领域 | 新增领域但未生成情境 | 新增时立即生成3-4个情境 |
| 学时调整后 | 调整学时但未调整情境数量 | 学时变化后重新计算情境数量 |

**情境数量计算公式**：

```
情境数量 = round(学习领域学时 ÷ 基准情境学时)
```

| 参数 | 取值 | 说明 |
|------|------|------|
| 基准情境学时 | 24学时 | 完成一个情境的平均学时 |
| 最少情境数 | 3个 | 每学习领域至少3个情境 |
| 最多情境数 | 8个 | 每学习领域最多8个情境 |

**计算示例**：
- 学习领域学时=72学时 → 情境数量=72÷24=3个
- 学习领域学时=96学时 → 情境数量=96÷24=4个
- 学习领域学时=128学时 → 情境数量=min(128÷24, 6)=5个

### 6.2 学习情境命名规则

**命名模式**：`[具体产品/设备] + [具体问题/任务] + [工作动词]`

**具体化原则**：
- 使用具体的产品型号或设备名称
- 描述具体的工作场景或问题
- 明确具体的工作内容

**命名示例**：

| 学习领域 | 情境命名示例 |
|----------|--------------|
| 汽车发动机检修 | 丰田卡罗拉发动机怠速不稳的检测与排除 |
| 网络设备配置 | 小型企业办公网络设备的基本配置 |
| 服装设计基础 | 春季休闲连衣裙款式设计 |
| 服装缝制工艺 | 男式衬衫领部缝制加工 |

### 6.3 情境学时分配指导原则

> **核心思想**：情境学时应根据**情境的教学价值**和**学生认知需求**分配，而非均分。

#### 6.3.1 情境学时分配依据

**1. 情境复杂度差异**

| 情境类型 | 特征 | 学时建议 | 示例 |
|----------|------|----------|------|
| **基础情境** | 简单案例，建立概念和方法 | 较少（16-24学时）| 汽车日常保养、网络基础配置 |
| **核心情境** | 典型案例，掌握核心技能 | 较多（24-32学时）| 发动机故障诊断、网络故障排除 |
| **拓展情境** | 复杂案例，综合应用能力 | 适中（16-24学时）| 新能源汽车维修、网络安全配置 |

**判断方法**：
- 分析情境涉及的**知识点数量**
- 评估情境的**操作复杂度**
- 考虑情境的**创新性要求**

**2. 教学阶段需求**

| 教学阶段 | 目的 | 学时特点 |
|----------|------|----------|
| **入门阶段**（情境1-2）| 建立基础概念和操作规范 | 学时可略多，便于夯实基础 |
| **发展阶段**（情境3-4）| 技能提升和应用拓展 | 学时适中，强调反复练习 |
| **综合阶段**（情境5+）| 能力综合和创新应用 | 学时可略少，学生已具备基础 |

**3. 学生认知负荷**

```
认知负荷判断：
├─ 新知识比例高 → 学时增加（需要更多理解和消化时间）
├─ 知识迁移为主 → 学时适中（利用已有基础）
└─ 综合应用为主 → 学时可减少（学生已掌握核心技能）
```

#### 6.3.2 情境学时分配流程

```
步骤1：评估每个情境的复杂度
       ├─ 分析知识点数量和难度
       ├─ 评估操作步骤复杂度
       └─ 考虑创新性要求
       
步骤2：确定情境在教学阶段中的位置
       ├─ 入门情境 → 学时可增加10-20%
       ├─ 发展情境 → 学时保持基准
       └─ 综合情境 → 学时可减少10-15%
       
步骤3：根据认知负荷调整
       ├─ 新知识多 → 学时增加
       ├─ 知识迁移 → 学时适中
       └─ 综合应用 → 学时减少
       
步骤4：验证情境学时合理性
       ├─ 单情境不低于16学时（教学完整性）
       └─ 单情境不高于32学时（学生承受能力）
```

#### 6.3.3 特殊情况处理

| 特殊情况 | 判断依据 | 处理方式 |
|----------|----------|----------|
| **情境学时<16学时** | 内容过于简单或情境划分过细 | 合并相邻情境，或增加情境深度 |
| **情境学时>32学时** | 内容过于复杂或认知负荷过高 | 拆分为两个情境，或简化情境范围 |
| **情境学时差异过大** | 分配不合理，学生学习节奏不稳定 | 重新评估各情境复杂度，调整分配 |

#### 6.3.4 情境学时范围参考

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| **最小情境学时** | 16学时 | 确保教学完整性和学生理解 |
| **最大情境学时** | 32学时 | 控制认知负荷，避免疲劳 |
| **推荐情境学时** | 24学时 | 平衡教学效率和学生学习效果 |

## 七、总学时分配与验证

### 7.1 总学时目标

| 教育层次 | 最低总学时 | 建议总学时 |
|----------|------------|------------|
| 中职 | 1200学时 | 1200-1400学时 |
| 高职 | 1600学时 | 1600-1800学时 |
| 职教本科 | 1600学时 | 1800-2000学时 |

### 7.2 学时分配验证与调整流程

> **核心思想**：学时验证应基于**教学合理性**分析，而非仅计算数值是否达标。

#### 7.2.1 学时合理性验证维度

**维度1：领域学时与任务复杂度匹配**

| 验证要点 | 检查方法 | 不匹配时的处理 |
|----------|----------|----------------|
| 简单任务领域学时过多 | 分析任务决策节点数 | 减少学时至合理范围 |
| 复杂任务领域学时过少 | 分析信息处理深度 | 增加学时以满足训练需求 |
| 领域内任务难度跨度大 | 检查是否存在"跳级" | 考虑拆分学习领域 |

**维度2：领域学时与教学方式匹配**

| 教学方式 | 合理学时范围 | 不匹配时的处理 |
|----------|-------------|----------------|
| 项目教学 | 64-144学时 | 学时<64时增加或调整教学方式 |
| 理实一体 | 48-120学时 | 学时<48时增加或调整教学方式 |
| 讲授式 | 32-72学时 | 学时>72时考虑增加实践环节 |

**维度3：总学时与教育层次匹配**

| 教育层次 | 最低总学时 | 验证重点 |
|----------|------------|----------|
| 中职 | 1200学时 | 实践学时占比是否≥60% |
| 高职 | 1600学时 | 实践学时占比是否50-60% |
| 职教本科 | 1800学时 | 理论学时占比是否≥40% |

#### 7.2.2 学时验证流程

```
步骤1：逐领域验证学时合理性
       ├─ 检查领域学时与任务复杂度是否匹配
       ├─ 检查领域学时与教学方式是否匹配
       └─ 检查领域学时是否在合理范围（48-144）
       
步骤2：汇总验证总学时
       ├─ 计算当前总学时 = Σ(各学习领域学时)
       ├─ 对比教育层次最低要求
       └─ 检查理论与实践比例是否合理
       
步骤3：识别学时问题类型
       ├─ 总学时不足 → 类型A（需增加领域或学时）
       ├─ 单领域异常 → 类型B（需调整该领域学时）
       ├─ 比例不合理 → 类型C（需调整理论与实践比例）
       └─ 总体合理 → 验证通过
       
步骤4：根据问题类型选择调整策略
```

#### 7.2.3 学时调整策略选择

**策略A：总学时不足的调整**

| 差距大小 | 推荐策略 | 具体操作 |
|----------|----------|----------|
| 差距<100学时 | 提高核心领域学时 | 核心领域各增加16-32学时 |
| 差距100-200学时 | 增加拓展学习领域 | 新增1个拓展领域（48-72学时）|
| 差距>200学时 | 综合调整 | 增加拓展领域 + 提高核心领域学时 |

**策略B：单领域学时异常的调整**

| 异常类型 | 判断依据 | 调整方式 |
|----------|----------|----------|
| 领域学时过多（>144）| 任务复杂度评估偏低 | 考虑拆分为两个领域，或重新评估任务复杂度 |
| 领域学时过少（<48）| 任务复杂度评估偏高 | 考虑合并相邻领域，或重新评估任务复杂度 |
| 领域学时与教学方式不匹配 | 项目教学但学时<64 | 增加学时或调整教学方式 |

**策略C：理论与实践比例不合理的调整**

| 问题类型 | 调整重点 | 调整方式 |
|----------|----------|----------|
| 实践学时不足（中职实践<60%）| 增加实践环节 | 核心操作类领域增加实践学时 |
| 理论学时不足（本科理论<40%）| 增加理论环节 | 综合分析类领域增加理论学时 |
| 比例失衡严重 | 综合调整 | 重新规划各领域的理论与实践分配 |

#### 7.2.4 学时调整优先级

| 优先级 | 调整方式 | 适用场景 | 调整幅度 |
|--------|----------|----------|----------|
| **1** | 增加学习领域数量 | 学习领域数量低于最低要求（中职<8，高职<10）| 新增1-2个领域（48-72学时）|
| **2** | 提高核心领域学时 | 核心技能领域学时偏低（<48）| 增加16-32学时 |
| **3** | 添加拓展学习领域 | 总学时差距较大（>200学时）| 新增拓展领域（48-72学时）|
| **4** | 调整情境学时 | 总学时差距较小（<50学时）| 每情境±4-8学时 |

### 7.3 学时调整优先级

| 优先级 | 调整方式 | 适用场景 |
|--------|----------|----------|
| 1 | 增加学习领域数量 | 学习领域数量低于最低要求 |
| 2 | 提高核心领域学时 | 核心技能领域学时偏低 |
| 3 | 添加拓展学习领域 | 总学时差距较大(>200学时) |
| 4 | 调整情境学时 | 微调总学时(差距<50学时) |

## 附录A：行业专家校验说明

> **重要提示**：行业专家校验是工作过程系统化课程开发方法的**核心环节**。在条件允许的情况下，建议邀请行业专家参与职业分析报告的编制和审阅。

### A.1 专家校验的重要性

工作过程系统化课程开发方法强调**行业专家参与**，原因如下：

| 参与环节 | 专家价值 | 缺失专家的风险 |
|----------|----------|----------------|
| 典型工作任务确定 | 确认任务是否反映当前行业实际 | 任务可能过时或不完整 |
| 行动领域划分 | 评估领域边界是否符合行业实践 | 领域划分可能与企业实际脱节 |
| 学时估算 | 提供真实岗位培训周期参考 | 学时可能偏高或偏低 |
| 能力描述 | 确认能力要求是否匹配岗位需求 | 能力描述可能过于理论化 |

### A.2 建议邀请的专家类型

| 专家类型 | 推荐人选 | 参与价值 |
|----------|----------|----------|
| **企业技术骨干** | 一线工程师、技师、主管 | 了解实际工作任务和技能要求 |
| **企业人力资源管理者** | 招聘经理、培训主管 | 了解岗位能力和职业发展路径 |
| **行业协会专家** | 行业协会技术委员、标准制定者 | 了解行业发展趋势和标准规范 |
| **职业院校资深教师** | 专业带头人、骨干教师 | 了解教学可行性和学生认知特点 |

### A.3 专家校验建议流程

```
推荐流程（条件允许时）：

阶段一：典型工作任务确定后（步骤9后）
├─ 邀请企业技术骨干审阅任务列表
├─ 确认：任务是否反映当前行业实际？
├─ 确认：任务难度划分是否合理？
├─ 确认：任务是否有遗漏？
└─ 专家签字确认或提出修改意见

阶段二：行动领域划分后（步骤10后）
├─ 邀请企业HR和行业专家审阅领域划分
├─ 确认：领域边界是否符合企业岗位划分？
├─ 确认：能力等级递进是否匹配职业发展路径？
└─ 专家签字确认或提出修改意见

阶段三：学时估算前（步骤12前）
├─ 邀请企业培训主管提供参考
├─ 了解：企业新员工培训周期？
├─ 了解：岗位技能熟练所需时间？
└─ 参考企业培训周期调整学时

阶段四：报告完成后（步骤17后）
├─ 邀请职业院校教师审阅教学可行性
├─ 确认：学时分配是否符合教学实际？
├─ 确认：情境设计是否具备可操作性？
└─ 专家签字确认或提出修改意见
```

### A.4 无法邀请专家时的替代措施

当无法邀请行业专家参与时，建议采取以下替代措施：

| 替代措施 | 具体操作 | 效果评估 |
|----------|----------|----------|
| **查阅行业报告** | 从行业协会、政府网站获取行业分析报告 | 了解行业趋势，但时效性可能滞后 |
| **参考职业大典2022版** | 确认任务与职业大典定义一致 | 保证规范性，但可能缺乏行业最新动态 |
| **对比类似专业报告** | 参考已发布的同类专业职业分析报告 | 借鉴成熟经验，但需注意专业差异 |
| **报告标注说明** | 在报告中明确标注"未经行业专家校验" | 提醒用户后续补充专家评审 |
