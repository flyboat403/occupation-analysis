## 步骤1：检索专业教学标准

### 数据源优先级

```
┌─────────────────────────────────────────────────────────────┐
│  1. 从 moe_pdfs_final.json 查找专业                         │
│     ↓                                                       │
│  2. 找到？                                                   │
│     ├─ YES → 获取 PDF URL                                   │
│     │         ↓                                             │
│     │    有 PDF URL？                                        │
│     │    ├─ YES → 下载并解析PDF → 返回完整内容             │
│     │    └─ NO  → 返回基本信息（无PDF URL）               │
│     │                                                       │
│  3. 未找到？ → 从 moe_pdfs_md 目录解析MD文件 🔄             │
└─────────────────────────────────────────────────────────────┘
```

### 数据源说明

| 优先级 | 数据源 | 内容 | 状态 |
|--------|--------|------|------|
| 1 | `moe_pdfs_final.json` | 专业代码、名称、PDF URL | 可用 |
| 2 | PDF文档解析 | 完整专业教学标准内容 | 可用（需安装pdfplumber或pypdf）|
| 3 | `moe_pdfs_md/*.md` | 已解析的MD文档 | 可用（备用）|

### 执行命令

```bash
# 完整检索（包含PDF解析）
python scripts/search_major.py --major "专业名称" --level "教育层次" --output temp/major_info.json

# 仅获取基本信息（不解析PDF）
python scripts/search_major.py --major "专业名称" --level "教育层次" --no-pdf --output temp/major_info.json
```

### 输出字段

| 字段 | 说明 | 来源 |
|------|------|------|
| `major_code` | 专业代码 | 必需 |
| `major_name` | 专业名称 | 必需 |
| `education_level` | 教育层次 | 必需 |
| `pdf_url` | PDF文档链接 | moe_pdfs_final.json |
| `pdf_parsed` | PDF是否已解析 | 布尔值 |
| `career_orientation` | 职业面向 | PDF解析或MD文件 |
| `training_goal` | 培养目标 | PDF解析或MD文件 |
| `main_courses` | 主要课程 | PDF解析（可选）|
| `data_source` | 数据来源 | primary/fallback |

## 边缘场景详细处理

### 步骤1边缘场景：PDF解析失败

**诊断检查**：

```bash
# 检查PDF URL是否有效
curl -I "https://example.com/path/to.pdf"

# 检查本地缓存
ls -la temp/pdf_cache/

# 检查备选数据源
ls assets/moe_pdfs_md/ | grep "专业代码"
```

**Fallback 方案**：

| 场景 | 命令 |
|------|------|
| PDF下载失败 | `python scripts/search_major.py --major "专业名称" --no-pdf` |
| 使用备选MD文档 | Agent直接读取 `assets/moe_pdfs_md/` 对应文件 |

### 步骤2边缘场景：职业大典格式异常

**常见异常及修复**：

```markdown
1. 职业代码格式错误
   - 错误示例：`4120201`（缺少分隔符）
   - 正确格式：`4-12-02-01`
   - 修复：手动补充分隔符

2. 职业定义缺失
   - 处理：从职业名称推断定义
   - 或从相近职业复制定义模板

3. 任务列表为空
   - 处理：从ESCO/O*NET对应职业反推任务
   - 示例：若ESCO有10个任务，翻译为中文后使用
```

**验证脚本**：

```python
# 验证职业信息完整性
import json
with open('temp/occupation_info.json', 'r', encoding='utf-8') as f:
    data = json.load(f)
    
for occ in data.get('occupations', []):
    issues = []
    if not occ.get('code'):
        issues.append("缺少职业代码")
    if not occ.get('definition'):
        issues.append("缺少职业定义")
    if not occ.get('tasks'):
        issues.append("缺少任务列表")
    
    if issues:
        print(f"[WARNING] {occ.get('name', '未知职业')}: {', '.join(issues)}")
```

### 步骤4边缘场景：映射低置信度处理

**低置信度判断标准**：

| 因素 | 权重 | 评分标准 |
|------|------|----------|
| 职业定义相似度 | 40% | 高度相似=40，部分相似=20，差异大=0 |
| 任务匹配度 | 40% | >70%=40，50-70%=20，<50%=0 |
| 行业领域一致性 | 20% | 同行业=20，相关行业=10，不同行业=0 |

**置信度计算**：
- 总分≥60 → `high`
- 总分40-59 → `medium`
- 总分<40 → `low`

**低置信度处理模板**：

```
【映射置信度警告】

职业名称：{中文名称}
职业代码：{中国代码}

推断结果：
- ESCO：{英文名称} ({代码}) - 置信度: {low/medium}
- O*NET：{英文名称} ({代码}) - 置信度: {low/medium}

原因分析：{具体差异说明}

建议操作：
1. 确认 - 接受当前映射继续
2. 修改 - 手动指定ESCO/O*NET代码
3. 跳过 - 不使用国际数据，仅用中国职业大典

请选择：
```

### 步骤5边缘场景：文档完全缺失

**缺失处理策略**：

```
文档缺失判断逻辑：
├─ ESCO文档存在 且 O*NET文档存在 → 正常使用
├─ 仅ESCO文档存在 → 使用ESCO + 职业大典任务
├─ 仅O*NET文档存在 → 使用O*NET + 职业大典任务
└─ 两者都缺失 → 仅使用中国职业大典，报告标注

报告标注格式：
> **数据完整性说明**：部分职业的国际参考数据（ESCO/O*NET）缺失，
> 本报告基于中国职业分类大典（2022版）编制。
```

**用户确认面板**：

```
【文档缺失警告】

以下职业的国际文档缺失：

┌──────────────────────────────────────────────────────────────────┐
│ 职业名称           │ ESCO文档 │ O*NET文档 │ 影响程度           │
├──────────────────────────────────────────────────────────────────┤
│ 新能源汽车维修工   │ ⚠️ 部分  │ ✅ 存在   │ 技能分析可能不完整 │
│ 汽车美容师         │ ❌ 缺失  │ ❌ 缺失   │ 仅使用中国职业大典 │
└──────────────────────────────────────────────────────────────────┘

影响说明：
- ESCO文档缺失：技能和知识描述可能不完整
- O*NET文档缺失：工作任务和能力描述可能不完整
- 两者都缺失：将仅使用中国职业大典数据

是否继续？
- 回复"确认"继续（使用已有数据）
- 回复"修改"调整职业映射
- 回复"取消"终止本次分析
```

### 自动恢复机制

```
错误类型判断：
├─ 可恢复错误（网络超时、格式异常）
│   └─ 自动尝试Fallback方案
├─ 需用户确认错误（文档缺失、映射低置信度）
│   └─ 展示警告面板，等待用户决策
└─ 不可恢复错误（关键数据文件丢失）
    └─ 终止流程，报告错误原因和修复建议
```
