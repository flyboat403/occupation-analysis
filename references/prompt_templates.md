>

## 1. 职业大典解析提示词模板

用于步骤3：大模型语义解析职业大典（分块优化版）。

### 完整模板

```
请从以下职业大典内容中检索与目标职业相关的信息，并以结构化格式输出：

【职业大典内容】（已按需加载对应大类）
{content}

【目标职业】
职业代码：{职业代码}
职业名称：{职业名称}

请提取以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 职业名称 | 标准名称 | 汽车维修工 |
| 职业代码 | 国家职业分类代码 | 4-12-02-01 |
| 职业定义 | 工作内容和特点描述 | 使用工具、量具和仪器... |
| 主要工作任务 | 核心任务列表（JSON数组） | ["任务1", "任务2", ...] |

输出格式要求：
1. 使用JSON格式输出
2. 每个职业一个对象，包含上述所有字段
3. 主要工作任务必须是字符串数组
4. 保持原文的专业术语准确性

示例输出：
```json
{
  "occupations": [
    {
      "职业名称": "汽车维修工",
      "职业代码": "4-12-02-01",
      "职业定义": "使用工具、量具和仪器设备，对汽车进行维护、修理和调试的人员",
      "主要工作任务": [
        "进行汽车定期维护和保养",
        "诊断汽车故障原因",
        "更换或修复损坏零部件",
        "调试和检验维修质量"
      ]
    }
  ]
}
```
```

### 简化模板（单职业提取）

```
我已为您加载了第{X}大类（{大类名称}）的职业大典内容，约{Y}K字符。

请从以下内容中提取职业代码 {职业代码}（{职业名称}）的信息：

[职业大典第{X}大类内容...]

按照指定格式输出JSON结构：
{
  "职业名称": "...",
  "职业代码": "...",
  "职业定义": "...",
  "主要工作任务": [...]
}
```

## 2. IMA 知识库智能检索提示词模板

用于步骤4：整合ESCO和O*NET数据。

### ESCO/O*NET 职业匹配检索

```
请从知识库检索与以下职业高度相似的occupation：

输入职业信息：
- 中文名称：{职业名称}
- 中国职业代码：{职业代码}
- 职业定义：{职业定义}

请检索并返回匹配度最高的occupation，包含以下字段：

| 字段 | 说明 |
|------|------|
| 职业名称 | ESCO/O*NET标准英文名称 |
| 职业代码 | ISCO代码 或 SOC代码 |
| 职业定义 | 英文职业描述 |
| 主要工作任务 | 核心任务列表（英中双语） |
| 必备技能 | Essential Skills |
| 可选技能 | Optional Skills |
| 知识要求 | Knowledge要求 |
| 工具技术 | Tools & Technology |
| 工作环境 | Work Context |
| 能力要求 | Abilities |

输出格式：md表格，包含上述所有字段，且主要工作任务需提供中英文对照。
```

### 物联网职业专用检索模板

```
请从 ESCO 知识库检索与以下物联网职业相关的 occupations：

【物联网职业】
- 中文名称：{职业名称}
- 中国职业代码：{职业代码}

【检索策略】
1. 使用关键词组合：IoT、Internet of Things、Embedded Systems、Sensor Networks、Smart Devices
2. 检索相关的 ESCO occupation groups
3. 返回匹配度最高的 3-5 个 occupations

【输出字段】
| 字段 | 说明 |
|------|------|
| ESCO URI | occupation URI |
| 职业名称 | 英文名称 |
| ISCO 代码 | 4位ISCO代码 |
| 技能要求 | essentialSkills + optionalSkills |
| 知识要求 | knowledge |
| 工作任务 | workTasks |

输出格式：JSON数组，每个occupation包含上述所有字段。
```

## 3. 数据整合分析提示词模板

用于步骤5：数据整合与分析。

### 典型工作任务提炼

```
请从以下职业的工作任务中提炼典型工作任务：

【原始数据】
{整合的职业数据JSON}

【提炼规则】
1. 任务难度符合教育层次（中职→操作为主，高职→技术应用，本科→管理优化）
2. 任务之间不重复，按工作逻辑排序
3. 每个典型任务可对应多个岗位的具体任务
4. 任务名称采用动词开头，如"诊断...故障"、"维护...设备"

【输出格式】
```json
{
  "typical_tasks": [
    {
      "task_id": "T001",
      "task_name": "典型任务名称",
      "related_jobs": ["岗位1", "岗位2"],
      "difficulty": "中等/复杂/简单",
      "description": "任务描述"
    }
  ]
}
```
```

### 行动领域划分

```
请将以下典型工作任务划分为行动领域：

【典型工作任务列表】
{任务列表JSON}

【划分规则】
1. 按工作对象相似性归类（如：处理相同设备、相同系统）
2. 每个行动领域包含 3-8 个典型任务
3. 行动领域名称反映工作性质，如"汽车动力系统维护"
4. 考虑任务难度梯度，同一领域内应有难度递进

【输出格式】
```json
{
  "action_domains": [
    {
      "domain_id": "AD001",
      "domain_name": "行动领域名称",
      "tasks": ["T001", "T002", "T003"],
      "description": "领域描述"
    }
  ]
}
```
```

### 学习领域转换

```
请将以下行动领域转换为学习领域（课程）：

【行动领域列表】
{行动领域JSON}

【转换依据】
1. 工作对象相似性：处理相同对象的任务归为一类课程
2. 任务难度梯度：从简单到复杂，符合学习认知规律
3. 工作逻辑顺序：按实际工作流程排列课程顺序
4. 学时合理性：每门课程参考学时 48-96 学时

【教育层次适配】
- 中职：侧重操作技能，课程名称体现"操作"、"维护"
- 高职：侧重技术应用，课程名称体现"应用"、"诊断"
- 职教本科：侧重系统优化，课程名称体现"设计"、"管理"

【输出格式】
```json
{
  "learning_domains": [
    {
      "course_id": "C001",
      "course_name": "课程名称",
      "action_domain": "AD001",
      "hours": 64,
      "objectives": ["目标1", "目标2"],
      "content_outline": ["内容1", "内容2"]
    }
  ]
}
```
```

用于步骤6：生成Word文档。

### 调用 docx skill 模板

```

【文档结构】
一、工作任务分析
- 表1-n: 各岗位岗位描述及典型工作任务

二、确定典型工作任务

三、典型工作任务分析
- 表3-n: 各典型工作任务分析表

四、归类、划分行动领域

五、行动领域分析
- 表5-n: 各行动领域职业能力分析

六、学习领域分析
- 表10: 学习领域表
- 表11-n: 各课程学习领域描述表

七、学习情境设计
- 表12: 学习领域学习情境表

【数据来源】
- 专业信息：{专业代码} {专业名称} {教育层次}
- 职业面向：{从教学标准提取的职业}
- ESCO数据：{从IMA知识库获取}
- O*NET数据：{从IMA知识库获取}

【结构参考】references/document_template.json
【输出路径】output/{专业名称}_职业分析报告.docx
```

## 使用说明

### 模板变量说明

| 变量 | 来源 | 示例 |
|------|------|------|
| `{content}` | 职业大典分块加载结果 | 第6大类内容 |
| `{职业代码}` | 专业教学标准检索结果 | 6-22-02-01 |
| `{职业名称}` | 专业教学标准检索结果 | 汽车维修工 |
| `{教育层次}` | 用户输入 | 高等职业教育本科 |
| `{大类名称}` | 根据职业代码判断 | 生产制造及有关人员 |
| `{专业名称}` | 用户输入 | 新能源汽车运用与维修 |
| `{专业代码}` | 检索结果 | 460702 |

### 模板选择指南

| 场景 | 推荐模板 | 说明 |
|------|----------|------|
| 解析单个职业 | 简化模板（单职业提取） | 更精确、更高效 |
| 解析多个职业 | 完整模板 | 批量处理 |
| 物联网相关专业 | 物联网职业专用检索 | 优化关键词组合 |
| 通用职业检索 | ESCO/O*NET 职业匹配检索 | 标准检索流程 |
| 数据整合分析 | 典型工作任务提炼 → 行动领域划分 → 学习领域转换 | 按顺序使用 |
| 文档生成 | 调用 docx skill 模板 | 包含完整结构说明 |
| IMA 检索失败 | 错误处理重试模板 | 多策略重试 |
